# Docker Images Configuration for Frontier-CS
# ===========================================
#
# This file specifies which problems should use predefined Docker images
# instead of the default python:3.11-slim-trixie image.
#
# Format: problem_name=docker_image_name,gpu_support,dind
#
# Where gpu_support can be:
# - "gpu" or "true" or "1" for --gpus all
# - "nogpu" or "false" or "0" for no GPU support
# - Empty (defaults to no GPU support)
#
# Where dind can be:
# - "dind" or "docker" for Docker-in-Docker (mounts /var/run/docker.sock)
# - Empty (no Docker socket mounting)
#
# Benefits of predefined images:
# - Faster evaluation (no dependency installation)
# - Pre-configured environments (e.g., CUDA, Triton)
# - Consistent builds across different systems
